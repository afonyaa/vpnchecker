# vpnchecker

Браузерный индикатор доступности заблокированных в РФ ресурсов (Instagram, X):
показывает по каждому «доступен / недоступен». Если доступны — у тебя работает VPN.
Без бэкенда, TypeScript + нативные ES-модули, сборка через Vite.

## Запуск

```bash
pnpm install
pnpm dev        # дев-сервер Vite → http://localhost:5173
```

Прод-сборка, проверка типов и предпросмотр:

```bash
pnpm typecheck  # tsc --noEmit, проверка типов
pnpm build      # tsc --noEmit + сборка в dist/
pnpm preview    # отдать собранное
```

> ES-модули не грузятся с `file://` (CORS), поэтому открывать нужно через сервер
> (`pnpm dev`), а не двойным кликом по `index.html`.

## Структура

| Файл | Назначение |
|------|------------|
| `index.html` | разметка и стили, подключает `src/main.ts` |
| `src/types.ts` | общие типы (`Check`, `Status`, `Target`, `AppState`) |
| `src/config.ts` | список заблокированных ресурсов |
| `src/net.ts` | `reachable()` — проверка достижимости через `fetch` no-cors |
| `src/ip.ts` | `lookupIp()` — IP пользователя + страна (ipwho.is, фолбэк ipify) |
| `src/store.ts` | `createStore<T>()` — минимальный реактивный стор (состояние + подписки) |
| `src/pipeline.ts` | `PipelineStep` — база цепочки (Chain of Responsibility) |
| `src/steps.ts` | `IpStep`, `ChecksStep` — шаги: сначала IP, затем проверки |
| `src/view.ts` | `render(state)` — декларативная отрисовка списка проверок в DOM |
| `src/main.ts` | собирает состояние, запускает проверки, обновляет стор |

Поток данных однонаправленный: `main` запускает проверки → каждая обновляет `store` →
подписка вызывает `render` → `view` перерисовывает список. Проверки заполняются по мере
готовности: `проверяю…` → `доступен` / `недоступен`.

## Ограничения

Метод — эвристика, не 100% гарантия. Подробный разбор подводных камней (CORS, DPI,
кэш, opaque-ответы, подмена страниц провайдером) — в [`PITFALLS.md`](./PITFALLS.md).
